1. Create `internal/formatting/markdown.go`:

   MarkdownV2 special chars that MUST be escaped in plain text regions:
   `_ * [ ] ( ) ~ ` > # + - = | { } . !`

   `func EscapeMarkdownV2(text string) string`:
   - Replace each special char with `\` + char
   - Characters to escape: `_`, `*`, `[`, `]`, `(`, `)`, `~`, `` ` ``, `>`, `#`, `+`, `-`, `=`, `|`, `{`, `}`, `.`, `!`
   - Use `strings.NewReplacer` for efficiency

   `func ConvertToMarkdownV2(text string) string`:
   Two-pass approach:
   1. Extract and save code blocks (``` ... ```) and inline code (` ... `) with placeholders
   2. In remaining text:
      - Headers: `^#{1,6}\s+(.+)$` -> `*$1*\n` (bold in MarkdownV2)
      - Bold: `**text**` -> `*text*` (MarkdownV2 bold)
      - Italic: `_text_` -> `_text_` (already MarkdownV2 italic)
      - Bullet lists: `^[-*] ` -> bullet char
      - Links: `[text](url)` -> `[text](url)` (same in MarkdownV2 but text must be escaped)
      - Escape all remaining special chars in plain text regions
   3. Restore code blocks: wrap in ` ``` ` (pre blocks don't need internal escaping in MarkdownV2)
   4. Restore inline code: wrap in `` ` `` (no internal escaping needed)
   5. Collapse multiple newlines

   `func StripMarkdown(text string) string`:
   - Remove all markdown formatting, return plain text
   - Remove `**`, `__`, `_`, `` ` ``, `#` headers, link syntax
   - This is the fallback when MarkdownV2 fails

   `func SplitMessage(text string, limit int) []string`:
   - Split at paragraph boundaries (last double-newline before limit)
   - If no paragraph boundary found, split at last newline before limit
   - If no newline found, split at limit
   - Per CONTEXT.md: "Message splitting at paragraph boundaries (last double-newline before 4096 char limit)"

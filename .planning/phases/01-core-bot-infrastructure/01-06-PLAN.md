   `type Bot struct`:
   - `bot *gotgbot.Bot`
   - `updater *ext.Updater`
   - `cfg *config.Config`
   - `store *session.SessionStore`
   - `persist *session.PersistenceManager`
   - `rateLimiter *security.ChannelRateLimiter`
   - `auditLog *audit.Logger`
   - `cancelFunc context.CancelFunc`
   - `wg sync.WaitGroup` — tracks active session worker goroutines

   `func New(cfg *config.Config) (*Bot, error)`:
   - Create `gotgbot.NewBot(cfg.TelegramToken, nil)`
   - Create `session.NewSessionStore()`
   - Create `session.NewPersistenceManager(filepath.Join(cfg.DataDir, "session-history.json"), config.MaxSessionHistory)`
   - Create `security.NewChannelRateLimiter(cfg.RateLimitRequests, cfg.RateLimitWindow)`
   - Create `audit.New(cfg.AuditLogPath)`
   - Log bot info: `zerolog.Info().Str("username", bot.User.Username).Str("claude_path", cfg.ClaudeCLIPath).Msg("Bot initialized")`

   `func (b *Bot) Start(ctx context.Context) error`:
   - Create cancellable context from parent
   - Restore sessions from persistence (load history, for each channel that has a saved session, create session in store, start worker goroutine with `b.wg.Add(1)`)
   - Create `ext.NewUpdater` with dispatcher
   - Register handlers via `b.registerHandlers(dispatcher)`
   - Call `updater.StartPolling(b.bot, ...)`
   - Log "Bot started, polling for updates"
   - Block on `ctx.Done()`

   `func (b *Bot) Stop()`:
   - Log "Shutting down..."
   - Call `b.cancelFunc()` to signal all session workers
   - Call `b.updater.Stop()` to stop polling
   - `b.wg.Wait()` — wait for all session workers to drain
   - Close audit log
   - Log "Shutdown complete"

3. Create `internal/bot/middleware.go`:

   `func (b *Bot) authMiddleware(next ext.Handler) ext.Handler`:
   - Extract userID from `ctx.EffectiveSender().Id()`
   - Call `security.IsAuthorized(userID, b.cfg.AllowedUsers)`
   - If not authorized: send reply "You're not authorized for this channel. Contact the bot admin." (per CONTEXT.md), log to audit, return
   - Call next handler

   `func (b *Bot) rateLimitMiddleware(next ext.Handler) ext.Handler`:
   - If `!b.cfg.RateLimitEnabled`: call next, return
   - Extract channelID from `ctx.EffectiveChat.Id`
   - Call `b.rateLimiter.Allow(channelID)`
   - If not allowed: send reply with format `"Rate limited. Try again in Xs."` where X is `delay.Seconds()` rounded (per CONTEXT.md), log to audit, return
   - Call next handler

1. Create `internal/handlers/streaming.go`:

   `type StreamingState struct`:
   - `bot *gotgbot.Bot`
   - `chatID int64`
   - `mu sync.Mutex`
   - `statusMsgID int64` — the ephemeral tool/thinking status message
   - `textSegments map[int]*TextSegment` — segment ID -> Telegram message + accumulated text
   - `lastEditTime time.Time`
   - `pendingText string` — text buffered waiting for throttle
   - `pendingTimer *time.Timer`

   `type TextSegment struct`:
   - `MessageID int64`
   - `Text string`

   `func NewStreamingState(bot *gotgbot.Bot, chatID int64) *StreamingState`

   `func CreateStatusCallback(ss *StreamingState) claude.StatusCallback`:
   Returns a function that handles each ClaudeEvent:

   For events with `event.Message != nil` (type "assistant"):
   - Iterate `event.Message.Content`:
     - `block.Type == "text"`: accumulate text into current segment. If throttle allows (500ms since last edit), call `editOrSendSegment()`. Otherwise buffer and set timer.
     - `block.Type == "thinking"`: update status message with "Thinking..."
     - `block.Type == "tool_use"`: call `FormatToolStatus(block.Name, block.Input)`, update status message

   For events with `event.Type == "result"`:
   - Flush any pending text
   - Delete status message if exists
   - Final segment edit with complete text

   `func (ss *StreamingState) editOrSendSegment(segmentID int, text string)`:
   - Convert text to MarkdownV2: `formatted := formatting.ConvertToMarkdownV2(text)`
   - If segment has no MessageID: `bot.SendMessage(chatID, formatted, &gotgbot.SendMessageOpts{ParseMode: "MarkdownV2"})`
     - If send fails with parse error: retry with `ParseMode: ""` and `formatting.StripMarkdown(text)` (plain text fallback per CONTEXT.md)
   - If segment has MessageID: `bot.EditMessageText(formatted, &gotgbot.EditMessageTextOpts{ChatID: chatID, MessageID: msgID, ParseMode: "MarkdownV2"})`
     - If edit fails with parse error: retry with `ParseMode: ""` and stripped text
   - If text exceeds `config.TelegramSafeLimit` (4000 chars): split via `formatting.SplitMessage`, send remaining as new segment

   `func (ss *StreamingState) updateStatusMessage(text string)`:
   - If statusMsgID == 0: send new message, save ID
   - Else: edit existing message (no parse mode — status is plain text with emoji)

   `func (ss *StreamingState) deleteStatusMessage()`:
   - If statusMsgID != 0: `bot.DeleteMessage(chatID, statusMsgID)`

   Typing indicator integration:
   `func (ss *StreamingState) SendThinkingMessage() int64`:
   - Send "Thinking..." message, return message ID
   - This message gets replaced by the first text segment or status update

2. Create `internal/handlers/text.go`:

   `func HandleText(b *gotgbot.Bot, ctx *ext.Context, store *session.SessionStore, cfg *config.Config, auditLog *audit.Logger, persist *session.PersistenceManager) error`:
   - Extract text from `ctx.EffectiveMessage.Text`
   - Extract chatID, userID
   - Log to audit: action="message", first 100 chars of text

   Interrupt handling (per CONTEXT.md and TypeScript):
   - If text starts with "!": strip prefix, get session, if running call `session.Stop()`, use stripped text as new message

   Command safety check:
   - Call `security.CheckCommandSafety(text, cfg.BlockedPatterns)`
   - If blocked: reply "Command blocked for safety: [pattern]", return

   Get or create session:
   - `sess := store.GetOrCreate(chatID, cfg.WorkingDir)`
   - If session has no worker running: start worker goroutine (with Bot's WaitGroup)

   Create streaming state and callback:
   - `ss := NewStreamingState(b, chatID)`
   - Send typing action: `b.SendChatAction(chatID, "typing", nil)`
   - Start typing indicator goroutine (4-second loop calling SendChatAction, stopped when streaming begins)

   Enqueue message:
   - Create `QueuedMessage{Text: text, ChatID: chatID, UserID: userID, Callback: func(...) { return CreateStatusCallback(ss) }}`
   - If `!sess.Enqueue(msg)`: reply "Queue full, please wait for the current query to finish."

   Error handling:
   - On Claude CLI error: truncate stderr to 200 chars, reply `"Claude error: [first 200 chars of stderr]"` (per CONTEXT.md)
   - On context limit: reply `"Session hit hard context limit and was cleared. Use /resume to restore a previous session."` (per CONTEXT.md)

   Persistence callback:
   - After query completes with sessionID: save to persistence manager with title = first 50 chars of prompt

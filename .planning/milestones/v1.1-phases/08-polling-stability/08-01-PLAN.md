Fix the long-polling timeout race condition by adding a per-request HTTP timeout that exceeds the getUpdates long-poll window.

Purpose: The current bot sets `GetUpdatesOpts.Timeout: 10` (10-second long-poll) but does not set a `RequestOpts.Timeout` on that request. The default HTTP client timeout can race the long-poll window, causing `context deadline exceeded` errors under normal idle conditions.

Output: Updated `internal/bot/bot.go` with `RequestOpts` added to `GetUpdatesOpts` so the HTTP client timeout is strictly greater than the long-poll window.
